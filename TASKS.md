## ⭐ 선행 리팩토링 (먼저 할 것)

지금 필기 캔버스 로직이 `LiveNoteView.tsx` 안에 섞여 있다. 아래 신규 기능 1·3이 **같은 드로잉 엔진을 PDF·캡쳐 슬라이드·빈 노트에서 공용**으로 쓰므로, 먼저 재사용 컴포넌트로 분리한다.

- `src/components/ink/InkCanvas.tsx` — 배경(투명/이미지/PDF 페이지) 위에 그리는 캔버스. props로 배경 source, 펜 설정, onStroke 콜백을 받는다.
- `src/components/ink/PenToolbar.tsx` — 펜 도구 UI(아래 기능 1).
- `src/lib/inkEngine.ts` — 펜 종류별 렌더 함수, 필압→선 굵기 매핑.
- 기존 `LiveNoteView`의 그리기 코드는 이 컴포넌트를 쓰도록 교체. 동기화(`pushDelta`)·원격 렌더(`renderInkSegment`)는 stroke 페이로드에 `type`/`pressure` 필드를 추가해 그대로 유지.

> 이렇게 한 번 만들면 기능 1(빈 노트·PDF), 기능 3(캡쳐 슬라이드)에서 재사용된다. 절대 3벌로 따로 만들지 말 것.

## A. 신규 기능

### A-1. 필기 UI 개편 — 더 크게 + 펜 종류/필압

**목표**: 현재 도구 바가 너무 작고 단순하다. 캔버스를 넓히고, 전문 필기앱 수준의 펜 도구를 제공한다.

**구현 가이드**
- **캔버스 확대**: 필기 영역이 화면 세로를 충분히 차지하도록. 경쟁 패널(AI 패널 등)은 접을 수 있게 하여 필기 공간 우선.
- **포인터 이벤트 전환**: `mousedown/move` 대신 `pointerdown/move/up`을 사용해 `e.pressure`(0~1, S펜·애플펜슬 지원)와 `e.pointerType`(pen/touch/mouse)을 받는다. 마우스(pressure=0)일 땐 속도 기반 굵기 또는 고정 굵기로 폴백.
- **펜 종류**(각각 렌더가 시각적으로 구분돼야 함):
  - 볼펜(pen): 불투명, 둥근 캡, 굵기 = base × (0.5 + pressure×gain)
  - 연필(pencil): 얇고 약간 거친 질감, 낮은 불투명도, 필압 영향 작게
  - 브러쉬(brush): 필압에 굵기 강하게 반응, 양끝 테이퍼(점점 가늘어짐)
  - 형광펜(highlighter): 넓고 굵기 일정, 알파 ~0.3, `globalCompositeOperation='multiply'`, 납작한 캡, 색은 형광 계열 몇 가지로 제한
  - 지우개(eraser): `destination-out`
- **펜 도구 팝오버**: 펜 아이콘을 누르면(또는 활성 펜 재탭) 팝오버가 뜨고 그 안에 ▸색상 팔레트 ▸굵기 슬라이더 ▸**필압 감도 슬라이더**(낮음↔높음, 내부적으로 pressure→굵기 gain 곡선을 조절: 높음=약한 힘에도 굵게, 낮음=세게 눌러야 굵게).
- 펜 모델: `{ type, color, baseWidth, pressureGain, opacity }` 로 관리하고 `inkEngine.ts`가 type별로 렌더.

**완료 기준**: 5종 펜이 시각적으로 구분되고, 필압 감도 슬라이더를 바꾸면 같은 힘에도 선 굵기 반응이 달라진다. S펜/애플펜슬에서 필압이 실제 반영된다.

### A-2. 실제 강의 녹음 → Whisper 전사 + 전사 패널(상시/탭 토글)

**목표**: 더미 요약 대신 **실제 음성 전사**가 녹음 중 실시간으로 쌓이고, 그 텍스트를 ①상시 표시 ②탭으로 접기/펼치기 둘 다 지원.

**전사 엔진 선택**(우선순위 순, 데모 친화 기준)
1. **온디바이스(권장)**: `@huggingface/transformers`(transformers.js)의 Whisper(`Xenova/whisper-base` 또는 `whisper-tiny`)를 WASM/WebGPU로 브라우저에서 실행. 키 불필요·프라이버시(앱의 NPU/온디바이스 콘셉트와 일치). 모델은 최초 녹음 시 lazy-load + 로딩 표시.
2. (폴백·가벼움) 브라우저 `webkitSpeechRecognition`로 실시간 캡션 — Chrome 한정, 정확도 낮음.
3. (정확도 최우선·선택) OpenAI Whisper API — `.env`의 키가 있을 때만, 서버/프록시 경유. 키 없으면 1번으로.

**구현 가이드**
- MediaRecorder는 이미 1초 청크(`recorder.start(1000)`)로 녹음 중. 청크를 ~5초 윈도로 모아 Whisper에 넣고, 결과 텍스트를 타임스탬프와 함께 전사 목록에 append.
- 처리 중 표시("전사 중…"), 줄마다 `[mm:ss] 텍스트`.
- **전사 패널 UI**: 우측(또는 하단) 도킹 패널. 측면에 **탭 핸들**을 둬 클릭으로 접기/펼치기. 열림/닫힘 상태는 `preferences.ts`에 영속(이미 패턴 있음).
- (보너스, 여유 시) 전사 줄을 리플레이의 잉크-오디오 매핑과 연결해 줄 클릭 시 해당 음성 시점으로 점프.

**완료 기준**: 실제로 말한 내용이 패널에 한국어로 전사돼 쌓이고, 탭으로 패널을 닫았다 열 수 있으며, 그 상태가 유지된다. 키 없이 온디바이스로 동작.

### A-3. 캡쳐된 슬라이드 위에 바로 필기

**목표**: 강의 판서 캡쳐(`LectureCapture.tsx`)는 잘 동작한다. 캡쳐된 슬라이드 위에 **즉시 필기**할 수 있게 한다.

**구현 가이드**
- '캡쳐된 슬라이드' 갤러리의 썸네일을 누르면 편집 뷰(모달 또는 전체 화면)가 열린다.
- 편집 뷰는 **캡쳐 이미지를 배경으로 깔고 그 위에 `InkCanvas`(선행 리팩토링 산출물)를 오버레이**해서 A-1의 펜 도구로 그릴 수 있게 한다.
- 저장: 주석이 입혀진 슬라이드를 (배경+stroke 합성 이미지로) 저장하고 노트에 첨부 또는 갤러리에서 갱신. 삭제도 가능.

**완료 기준**: 캡쳐 슬라이드를 열어 A-1의 펜으로 필기하고 저장하면, 그 필기가 남은 채로 갤러리/노트에 보인다.

## B. 남은 우선순위 수정 (작은 작업, 먼저 끝내도 됨)

### B-1. [김지원] 녹음 저장 실패 경고 제거 → 로컬 안전 보관 안내
- 위치: `src/components/LiveNoteView.tsx`, `recorder.onstop` 내부(현재 124줄 `alert("Google Drive 저장에 실패했습니다…")`).
- 변경: Drive 토큰이 없거나(게스트/미연결) 업로드 실패 시 **alert 금지**. 대신 녹음 blob을 로컬 보관(state 또는 IndexedDB)하고 `showToastMsg("로컬에 안전하게 보관됨 · 연결 시 자동 동기화")` 같은 **안심 톤** 안내. 토큰이 있을 때만 Drive 업로드를 시도.
- 이유: 데이터 유실 불안이 높은 핵심 페르소나(김지원)에게 "방금 녹음이 실패"라는 경고는 신뢰를 통째로 깨뜨림. "저장 중…" 토스트 뒤에 "실패" 경고가 뜨는 신호 충돌도 제거.
- 완료 기준: 게스트로 들어가 녹음을 멈춰도 경고창이 뜨지 않고, "로컬 보관됨" 안내만 표시.

### B-2. [전원] PDF 단어 검색 크래시 수정
- 위치: `src/components/pdf/PdfAdvancedRenderer.tsx` 104줄 부근 `window.pdfjsLib.Util.transform(...)`.
- 변경: 파일 상단에서 이미 `pdfjsLib`로 import 중이므로 `window.pdfjsLib` → `pdfjsLib`로 교체. (런타임 크래시이며 타입 문제 아님.)
- 가능하면 `?url` 워커 import의 타입 선언도 `src/vite-env.d.ts`에 `declare module '*?url'`로 정리.
- 완료 기준: PDF에서 단어 검색 시 매칭 위치에 하이라이트가 뜨고 화면이 멈추지 않는다.

### B-3. [박철수] 기기 추가 페이월 톤 완화
- 위치: `src/components/SettingsView.tsx` 업그레이드 모달(현재 713줄 "기기 추가를 위해 계정 업그레이드가 필요합니다.").
- 변경: 강매성 '벽' 문구를 정보성으로. 예) "지금 무료로 2대를 쓰고 있어요. 3대 이상 연결은 Pro에서 가능합니다." + 닫기 부담 없이. 로그인 화면에서 약속한 "$0"과 충돌하지 않도록 결제 강요 인상 제거.
- 완료 기준: 기기 추가 시 거부감 없는 안내로 보이고, 무료 사용자가 막혔다는 느낌을 받지 않는다.
